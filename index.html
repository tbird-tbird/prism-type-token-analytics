Methodology: This analysis traces alias chains from all type-related variables in Prism Foundations (24 usage semantic primitives, 148 usage/type composite properties, and 182 comp/type properties) back to base type tokens in Prism Base, across all 32 theme modes — not just Default mode. A base token is counted as "referenced" if any usage or comp token in any mode resolves to it.

Observations

  • 30% of base type tokens are unreferenced across all modes (17 of 57). The 10 semantic font-size aliases account for the majority. Only 7 non-alias tokens are truly unused: font-size/5, font-size/100, font-size/130, letter-spacing/20, letter-spacing/40, line-height/0, and line-height/130.
  • Semantic font-size aliases are entirely unused — the 10 semantic naming tokens at the base level are never referenced in any mode because usage tokens alias directly to numeric steps.
  • Letter-spacing is more differentiated than it appears in Default — while Default mode collapses all three usage tokens to step 70, brand modes like Dash Plus and Wolt Courier Plus use a full spread across steps 0, 10, 30, and 50. Only steps 20 and 40 go completely unused.
  • "Plus" modes drive upper-range consumption — Dash Plus, Wolt Courier Plus, and Deliveroo Rider Plus modes use larger type scales (steps 35, 50, 80) that Default mode doesn’t touch. Mx Tablet and Merchant POS Plus extend even further into steps 70, 90, 120, 160, and 190.
  • All 5 font-weight tokens are consumed when considering all modes. light is used by Wolt Consumer and Merchant Next; semibold is used by Wolt, Merchant, and Deliveroo Merchant modes.
  • The type scale is well-utilized across modes — at 70% consumption (excluding semantic aliases), the base type token set is more efficiently consumed than it appeared in Default-only analysis.
