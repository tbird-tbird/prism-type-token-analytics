Methodology: This analysis traces alias chains from all 182 comp/type/* tokens back to usage-level tokens across all 32 theme modes. Usage tokens exist at two levels: 24 semantic primitives (e.g., usage/font-size/small) and 154 composites (e.g., usage/type/label/medium/strong/font-size). Comp tokens may reference either level. A usage token is counted as "referenced" if any comp token in any mode aliases to it.

Comp token aliasing patterns

Comp tokens reference usage tokens at two different levels, creating an inconsistent aliasing pattern across the system.

PatternComp tokensTarget layerStyle categories
Via compositesMajority of label, title compsusage/type/* compositeslabel (large, medium, small, x-small/strong), title (large only)
Direct to primitivesavatar, table, tag, some button variantsusage/font-*, usage/line-height/*, usage/letter-spacing/* primitivesMixed — bypasses composite layer
Inconsistency: Some comp tokens alias to usage/type/label/medium/strong/font-size (the composite), while others alias directly to usage/font-size/x-small (the semantic primitive). Both patterns coexist within the same component — for example, comp/type/tag/label/small/font-size references the composite but comp/type/tag/label/large/letter-spacing references the primitive.

Body, Display, Link composites (0 of 84 referenced)

All 42 body, 21 display, and 21 link composites have zero references from comp tokens.

Observations

  • All 24 usage semantic primitives are actively consumed across modes. There are no orphaned primitives, making this a well-utilized abstraction layer.
  • 74% of usage/type composites are unreferenced (114 of 154). The entire body (42), display (21), and link (21) categories have zero comp token consumers. These composites exist but serve no downstream aliasing purpose at the comp level.
  • Label composites drive the system — label styles account for 3,201 of 3,363 total composite references (95%). They are consumed by button, button-tab, button-toggle-group, field, banner, avatar, table, and tag components.
  • Inconsistent aliasing patterns exist — some comp tokens alias through the composite layer (usage/type/*) while others bypass it entirely and alias directly to semantic primitives (usage/font-*). This mixed pattern makes dependency tracing less predictable.
  • text-case and text-decoration are universally unreferenced — all 20 text-case and text-decoration composite tokens across every style category are never consumed by any comp token.
  • usage/font-weight/xstrong is mode-specific — it only applies to Consumer and Consumer Dark modes, used by display and title styles. All other modes use strong or default weights instead.
  • Letter-spacing primitives have limited mode coverage — even the most-used letter-spacing token (small) is absent from half the modes, suggesting many brand themes collapse letter-spacing differentiation.
